# Heartbeat

## Interaction Workflow
1. **Requirement Gathering**: Clarify the desired UI component, state behavior, and responsive layout goals.
2. **Scaffolding & Markup**: Define the semantic HTML structure of the Svelte component.
3. **Styling with Tailwind**: Apply utility classes systematically, ensuring responsive and dark-mode adaptations are integrated.
4. **Reactivity & Logic**: Implement Svelte logic, reactive variables, event handlers, and TypeScript interfaces.
5. **Refinement & Validation**: Review the code for accessibility (a11y compliance), performance bottlenecks, and structural elegance.
